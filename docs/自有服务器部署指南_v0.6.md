# 《听到了咩 / heard-sheep》自有服务器部署指南 v0.6

本指南用于把《听到了咩》部署到你自己的 Linux 服务器，并通过公网 HTTPS 在手机上访问与安装 PWA。

当前推荐两种部署方式：

- Docker / Docker Compose：更适合长期运行和迁移。
- Node.js + PM2 + Nginx：更适合直接在服务器上调试。

无论采用哪种方式，都请保留 `/sheep` basePath，不要在反向代理里剥掉 `/sheep` 路径。

## 1. 服务器准备

建议环境：

- Ubuntu 22.04 / Debian 12 或同级 Linux 服务器
- Node.js 22 LTS（PM2 方式需要）
- Docker + Docker Compose（Docker 方式需要）
- Nginx / Caddy / 宝塔反代均可
- 一个已解析到服务器的域名
- HTTPS 证书

PWA、Service Worker、手机麦克风权限在公网真实使用时应走 HTTPS。

## 2. 环境变量

在服务器上创建 `.env.production` 或 `.env.local`，不要提交到 GitHub。

```env
NEXT_PUBLIC_BASE_PATH=/sheep

AI_PROVIDER=deepseek
AI_ALLOW_MOCK_FALLBACK=true
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_TIMEOUT_MS=60000

XIAOMI_API_KEY=
XIAOMI_BASE_URL=https://api.xiaomimimo.com/v1
XIAOMI_TIMEOUT_MS=60000

VISION_PROVIDER=xiaomi-image
VISION_ALLOW_TEXT_FALLBACK=true
XIAOMI_IMAGE_MODEL=mimo-v2.5

ASR_PROVIDER=xiaomi-audio
ASR_ALLOW_MOCK_FALLBACK=true
XIAOMI_AUDIO_MODEL=mimo-v2.5
XIAOMI_AUDIO_INPUT_MODE=input_audio
```

完整变量参考根目录 `.env.example`。

## 3. Docker Compose 部署

在服务器上：

```bash
git clone https://github.com/Jamie-zhang1/heard-sheep.git
cd heard-sheep
cp .env.example .env.production
```

填写 `.env.production` 中的真实 Key，然后：

```bash
cp docker-compose.example.yml docker-compose.yml
docker compose up -d --build
```

检查：

```bash
docker compose ps
docker compose logs -f heard-sheep
curl -I http://127.0.0.1:3000/sheep
```

数据目录：

```text
./data:/app/data
```

这样可以把 MVP 的 JSON 数据文件保留在宿主机目录中。仍建议定期备份 `data/records.json`。

## 4. Node.js + PM2 部署

在服务器上：

```bash
git clone https://github.com/Jamie-zhang1/heard-sheep.git
cd heard-sheep
cp .env.example .env.local
npm ci
npm run build
npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save
```

检查：

```bash
pm2 status
pm2 logs heard-sheep
curl -I http://127.0.0.1:3000/sheep
```

更新代码：

```bash
git pull
npm ci
npm run build
pm2 restart heard-sheep
```

## 5. Nginx 反向代理

示例配置见：

```text
deploy/nginx/heard-sheep.conf.example
```

关键点：

- `proxy_pass http://127.0.0.1:3000;`
- 不要把 `/sheep` rewrite 掉。
- `client_max_body_size` 建议至少 `30m`，用于图片和音频上传。
- API 超时时间建议 120-180 秒，避免图片理解或音频转写被 Nginx 提前断开。

启用后访问：

```text
https://你的域名/sheep
```

根路径 `/` 会由 Next.js 重定向到 `/sheep`。

## 6. HTTPS 与 PWA 检查

手机真机测试前，确认：

- `https://你的域名/sheep` 能打开首页。
- `https://你的域名/sheep/manifest.json` 返回 200。
- `https://你的域名/sheep/sw.js` 返回 200。
- Chrome DevTools → Application → Manifest 能看到应用名与新版小羊图标。
- Chrome DevTools → Application → Service Workers 能看到 scope 为 `/sheep/`。
- `/sheep/api/*` 请求没有被 Service Worker 缓存。

## 7. 数据持久化与备份

当前仍是单用户 MVP，本地 JSON 文件为主：

```text
data/records.json
```

自有服务器比 Vercel 更适合保存这个文件，但它仍不是正式数据库。建议：

- Docker 部署时挂载 `./data:/app/data`。
- 定期备份 `data/records.json`。
- 正式长期使用前迁移到 PostgreSQL / SQLite 文件卷 / 对象存储。

## 8. 安全注意

- 不要把 `.env.local`、`.env.production`、真实 API Key 提交到 Git。
- 服务器防火墙只开放 80 / 443 / 必要 SSH 端口。
- Node 服务建议只监听内网端口，由 Nginx 反代。
- 图片和音频上传目前适合个人自测，不建议开放给不可信用户。
