# 《听到了咩 / heard-sheep》手机自测版部署指南 v0.6

## 1. 部署目标

本阶段目标是把当前 Next.js Web MVP 部署到公网 HTTPS 地址，方便在 iPhone / Android 手机上真实访问、添加到主屏幕，并持续回收录音、图片识别、文本分析、任务追踪等真实使用问题。

推荐部署形态：

- 平台：Vercel
- 访问路径：`/sheep`
- 展示形态：可安装 PWA
- 主要用途：个人手机真机自测，不是多用户生产服务

## 2. 部署前准备

请先确认：

- GitHub `main` 分支已同步最新代码。
- 本地 `npm run typecheck` 与 `npm run build` 通过。
- 已准备 DeepSeek 与 Xiaomi 的服务端 API Key。
- `.env.local` 不会提交到 GitHub。

### 数据存储限制

当前项目仍使用本地 JSON 文件存储 `data/records.json`。在 Vercel 这类 Serverless 环境中，运行时文件写入不适合作为长期持久化存储：

- 部署包内文件不可作为稳定数据库使用。
- 函数实例可能重启或迁移，历史数据不保证长期保存。
- 当前部署适合个人自测、演示和流程验证。
- 后续若要长期使用，应迁移到 SQLite 托管、Postgres、KV 或对象存储。

## 3. Vercel 环境变量

在 Vercel Project Settings → Environment Variables 中配置。不要使用真实 Key 写入文档或代码。

```env
NEXT_PUBLIC_BASE_PATH=/sheep

AI_PROVIDER=deepseek
AI_ALLOW_MOCK_FALLBACK=true
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_TIMEOUT_MS=60000

XIAOMI_API_KEY=
XIAOMI_BASE_URL=https://api.xiaomimimo.com/v1
XIAOMI_TIMEOUT_MS=60000

VISION_PROVIDER=xiaomi-image
VISION_ALLOW_TEXT_FALLBACK=true
XIAOMI_IMAGE_MODEL=mimo-v2.5

ASR_PROVIDER=xiaomi-audio
ASR_ALLOW_MOCK_FALLBACK=true
XIAOMI_AUDIO_MODEL=mimo-v2.5
XIAOMI_AUDIO_INPUT_MODE=input_audio
```

如需继续保留其他 Provider，可参考仓库根目录 `.env.example`。

## 4. Vercel 部署步骤

1. 登录 Vercel。
2. 点击 `Add New Project`。
3. Import Git Repository，选择 `heard-sheep`。
4. Framework Preset 保持 `Next.js`。
5. 填入上方环境变量。
6. 点击 `Deploy`。
7. 部署完成后访问：

```text
https://你的域名/sheep
```

根路径 `/` 会重定向到 `/sheep`，但正式分享建议直接使用 `/sheep` 地址。

## 5. 手机安装步骤

### iPhone Safari

1. 用 Safari 打开部署地址。
2. 点击底部分享按钮。
3. 选择“添加到主屏幕”。
4. 回到主屏幕，确认图标为新版小羊。
5. 从主屏幕图标打开，检查 standalone 模式布局。

### Android Chrome

1. 用 Chrome 打开部署地址。
2. 如果出现“安装应用”提示，点击安装。
3. 若未自动提示，点击右上角菜单。
4. 选择“安装应用”或“添加到主屏幕”。
5. 从主屏幕图标重新打开。

## 6. 部署后验收

至少检查：

- 首页能打开，路径为 `/sheep`。
- Manifest 中应用名、图标、主题色正确。
- Service Worker 已注册，且 `/sheep/api/*` 未被缓存。
- 新版小羊 favicon / 主屏幕图标显示正常。
- 首次录音可触发麦克风权限。
- 录音结束后能进入 Xiaomi 音频理解转写。
- 图片上传后能进入 Xiaomi 图片理解提取文字。
- 粘贴文本能进入 DeepSeek 分析。
- 结果页、任务详情、任务编辑、历史页可正常打开。
- PWA standalone 模式下底部导航不被系统手势区遮挡。
