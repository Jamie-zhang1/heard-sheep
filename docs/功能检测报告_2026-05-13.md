## 4. 核心流程状态

### 4.1 录音主流程

```text
首页 → 开始录音 → 结束录音 → 转写中 → 转写确认 → AI 分析中 → 结果页
```

状态：已实现。

实现说明：

- 使用 MediaRecorder 获取音频。
- 浏览器支持 Web Speech API 时，会优先尝试实时识别。
- 没有浏览器识别文本时，调用 `/sheep/api/transcribe`。
- 服务端转写当前是 mock ASR。
- 录音中支持暂停、继续、结束、重点标记。
- 录音少于 5 秒时出现短录音提示。

### 4.2 上传音频流程

```text
首页 → 上传音频 → 开始转写 → 转写确认 → AI 分析 → 结果页
```

状态：已实现。

限制：默认服务端转写结果为 mock；配置真实 OpenAI-compatible ASR 后可调用真实供应商。

### 4.3 粘贴转写稿流程

```text
首页 → 粘贴转写稿 → 示例文本 / 手动输入 → AI 分析 → 结果页
```

状态：已实现。

### 4.4 图片识别流程

```text
首页 → 识别图片 → 上传图片 → AI 分析 → 结果页
```

状态：入口和链路已实现。

限制：

- mock 模式不会真实 OCR。
- DeepSeek V4 Flash 当前不支持直接图片输入；图片入口已降级到“确认图片文字”流程，避免生成失败中断。
- 真实效果依赖配置的 LLM Provider 是否支持视觉输入。

## 7. AI / ASR 状态

### AI

- 默认 provider：`mimo`。
- 当前验证通过模型：`mimo-v2.5-pro`、`mimo-v2.5`。
- 当前推荐模型：`mimo-v2.5-pro`，建议 `MIMO_TIMEOUT_MS=60000`。
- 未配置 `MIMO_API_KEY` 时：使用 mock。
- 配置 MiMo 失败时：在允许 fallback 的情况下返回 `mock_fallback`。
- 输出结构由 Zod 校验。

### ASR

- 浏览器端：支持 Web Speech API 的浏览器会尝试实时识别。
- 服务端：`/api/transcribe` 已 provider 化，支持 OpenAI-compatible `/audio/transcriptions` provider。
- 当前默认配置仍为 mock，真实 ASR 需要配置 `ASR_API_KEY`、`ASR_BASE_URL` 和 `ASR_MODEL`。

## 9. 当前限制

- 服务端 ASR 默认仍是 mock，但已有 OpenAI-compatible provider。
- 图片识别不是独立 OCR 服务，当前 DeepSeek V4 Flash 不支持直接读图，已提供文字确认 fallback；后续需要接视觉模型或 OCR Provider。
- 日历联动、会员、评分、退出登录为 MVP 占位。
- 当前是本地单用户 MVP，没有账号体系和云端同步。
- 尚未配置 GitHub Actions 自动化检测。
