# 小羊视觉资产使用说明 v0.6

更新时间：2026-05-15

## 1. 升级目标

本轮将 `docs/各种图标` 中的新小羊设计整理为《听到了咩》的统一视觉资产来源：

- 新小羊主形象成为品牌默认 mascot；
- 旧版线框/旧贴图退出正式页面主使用；
- 等待、成功、错误、空状态、鼓励、轻社交等状态都有明确视觉映射；
- 页面不直接硬编码素材路径，统一通过 `lib/sheep-assets.ts` 与 `components/SheepVisual.tsx` 使用。

## 2. 素材分类

### 主品牌图

- `mascot`：标准正面小羊，用于首页品牌区、我的页头像、背景低透明装饰、PWA 图标来源。

### 正向反馈图

- `received` / `success`：收到，用于保存、识别、导出等完成反馈。
- `ok`：好的，用于轻量确认。
- `cheer`：冲鸭，用于任务完成、安装引导、鼓励用户继续推进。
- `thanks`：谢谢，用于反馈和导出完成后的感谢表达。

### 处理中 / 思考图

- `thinking`：省略号等待小羊，用于转写中、图片识别中、AI 分析中。
- `recording`：录音/转写处理中，当前复用 `thinking`。

### 错误图

- `error` / `sad`：委屈小羊，用于图片识别失败、转写失败、AI 生成失败。
- `cry`：更强失败态，保留但少用。
- `angry`：不满情绪，当前仅作为素材保留，不建议在核心流程频繁使用。

### 空状态图

- `empty` / `question`：问号小羊，用于暂无任务、暂无历史、搜索无结果、需要用户补充信息。

### 问候与轻社交图

- `goodMorning`：早呀，适合首次欢迎或早间问候。
- `goodNight`：晚安，仅建议夜间场景使用。
- `love` / `shy` / `hardwork` / `bye`：适合评分、帮助、结束提示等轻社交场景。

## 3. 产品使用场景

| 场景 | 推荐 variant | 当前落地 |
| --- | --- | --- |
| PWA / favicon / Apple touch icon | `mascot` | 已生成 192 / 512 / maskable 图标 |
| 首页品牌区 | `mascot` | 已替换旧小羊 |
| 我的页头像 | `mascot` | 已替换旧小羊 |
| 背景低透明装饰 | `floating` | 通过 `PhoneShell` 统一渲染 |
| 转写中 / 图片识别中 / AI 分析中 | `thinking` | `ProcessingOverlay` |
| 错误态 | `error` | `ErrorOverlay` |
| 空状态 | `empty` | `EmptyState` |
| 安装引导 | `cheer` | `PwaInstallPrompt` |
| 系统标签里的小羊 | `mascot` / `question` / `cheer` | `LabelPill` |

## 4. 不建议使用方式

- 不要把带文字贴纸直接用作 favicon 或 PWA 图标；
- 不要在每个卡片里都塞小羊，避免喧宾夺主；
- 不要把错误态使用“哈哈”“冲鸭”这类正向表情；
- 不要把大尺寸原图直接塞进首屏；
- 不要直接在页面中硬编码 `/brand/sheep/...` 路径，应通过 `SheepVisual` 使用。

## 5. 素材处理说明

原始素材位于本地 `docs/各种图标`。检查发现这些 PNG 文件没有 alpha 通道，棋盘格背景被烘进了图片中，因此正式页面不直接引用原图。

本轮生成的运行时资源位于：

```text
public/brand/sheep/
├─ mascot/
├─ stickers/
├─ ui/
└─ pwa/
```

其中：

- `mascot/` 与 `stickers/` 保留较高质量展示图；
- `ui/` 提供更轻量的小尺寸页面用图；
- `pwa/` 生成 PWA 图标源；
- `public/icons/` 提供 manifest 引用的 192 / 512 / maskable 图标。

## 6. 旧资源处理

旧版 `public/brand/sheep-*.png/svg/jpg` 已从正式资源中移除。当前页面主视觉统一来自新版 `public/brand/sheep/` 目录。
